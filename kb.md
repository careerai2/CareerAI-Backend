# Internship Document Formatting Guidelines

## Instructions
- Provide concise, actionable guidelines only.
- Focus only on the fields that are present in \'patched_fields\'.
- Do not provide instructions for untouched fields.
- Do not add filler or irrelevant content.
- Always specify formatting requirements for each field (capitalization, length, structure).

## Action Verbs (to use in work descriptions)
Developed, Implemented, Optimized, Engineered, Automated, Tested, Designed, Deployed,
Refactored, Collaborated, Researched, Analyzed, Documented

## Schema Requirements & Formatting Rules

### Company Name
- Use official registered name only.
- Avoid abbreviations unless globally recognized (e.g., IBM).
- Apply Title Case.
- Example: **Google LLC.**

### Company Description
- Limit to 1–2 lines max.
- Mention industry/domain + products/services.
- Avoid marketing fluff.
- Example: *A fintech startup providing AI-driven investment solutions.*

### Location
- Format: **City, Country.**
- Example: *Bengaluru, India.*

### Designation
- Write the exact internship title.
- Capitalize each word.
- Example: **Software Engineering Intern.**

### Designation Description
- Limit to 1–2 lines max.
- Clearly state role/responsibility.
- Example: *Assisted in developing backend APIs for internal tools.*

### Duration
- Format: **MMM YYYY – MMM YYYY** (or *Present* if ongoing).
- Example: *Jun 2024 – Aug 2024.*

### Internship Work Description Bullets
- Provide 3–5 bullets.
- Each bullet must:
  - Begin with an action verb (see list).
  - Mention tools/tech used.
  - Be max 20 words.
  - Include quantified results where possible.
- Example: *Developed a data pipeline using Python & SQL, reducing processing time by 30%.*

### Tools & Technologies
- Write as a comma-separated list.
- Example: *React, Node.js, PostgreSQL, Docker.*

### Impact
- Limit to 1–2 lines.
- Show measurable outcomes (numbers/percentages preferred).
- Example: *Improved query performance by 40%, reducing report generation time from 5s to 3s.*

### Team Collaboration
- Keep to 1 line.
- Mention team size or methodology.
- Example: *Collaborated in a 5-member team using Agile Scrum practices.*

### Learning Outcomes
- Limit to 1–2 lines.
- State skills or domain knowledge gained.
- Example: *Gained hands-on experience in microservices architecture and cloud deployment.*

# Position of Responsibility Document Formatting Guidelines

## Instructions
- Provide concise, actionable guidelines only.
- Focus only on the fields that are present in the `PositionOfResponsibility` schema.
- Do not provide instructions for untouched fields.
- Always specify formatting requirements for each field (capitalization, length, structure).
- Ensure consistency across all entries.

## Action Verbs (to use in responsibilities)
Led, Coordinated, Organized, Managed, Directed, Facilitated, Oversaw, Represented,
Initiated, Guided, Planned, Executed, Conducted, Supported

## Schema Requirements & Formatting Rules

### Role
- Write the exact position title.
- Capitalize each word.
- Example: **Event Coordinator.**

### Role Description
- Limit to 1–2 lines max.
- Clearly describe the purpose or scope of the role.
- Example: *Coordinated logistics and scheduling for technical workshops.*

### Organization
- Use the official registered name only.
- Avoid abbreviations unless globally recognized (e.g., IEEE).
- Apply Title Case.
- Example: **Institute of Electrical and Electronics Engineers (IEEE).**

### Organization Description
- Limit to 1–2 lines max.
- Mention the type of organization (club, NGO, association, etc.) and its focus area.
- Example: *A global association dedicated to advancing innovation and technology.*

### Location
- Format: **City, Country.**
- Example: *New Delhi, India.*

### Duration
- Format: **MMM YYYY – MMM YYYY** (or *Present* if ongoing).
- Example: *Jan 2023 – Dec 2023.*

### Responsibilities
- Provide 3–5 bullets.
- Each bullet must:
  - Begin with an action verb (see list).
  - Be max 20 words.
  - Highlight leadership, management, or organizational impact.
  - Include measurable outcomes where possible.
- Example: *Led a 10-member team to organize a national hackathon with 300+ participants.*
